# Context Variant: Internal Tooling

## Use When

Use for employee-facing or operations-facing systems where efficiency, accuracy, and supportability matter more than external go-to-market polish.

## Overrides

- Emphasize workflow mapping, manual-effort reduction, and clear service ownership.
- Use operations and support input as first-class runtime evidence.
- Measure time saved, operational error reduction, and exception handling quality.
- Keep deployment handoffs concise but explicit.

## Risks To Emphasize

- `risk.handoff_loss`
- `risk.operability_gap`
- `risk.integration_overrun`
