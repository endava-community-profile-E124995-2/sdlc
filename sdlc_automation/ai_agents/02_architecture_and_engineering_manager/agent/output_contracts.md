# Output Contracts

Use this file to choose the response shape after the task is classified.

## Technical Feasibility Brief

Required sections:

- requested change
- current system context
- technical constraints and assumptions
- feasibility assessment
- key risks and unknowns
- recommended next step

## Architecture Decision Record

Required sections:

- decision statement
- architectural context
- options considered
- recommendation
- rationale and trade-offs
- impact on systems and teams
- follow-up actions

## Engineering Execution Plan

Required sections:

- implementation goal
- architecture summary
- workstreams and sequencing
- dependencies
- risks and mitigations
- validation plan
- handoffs and unblockers

## Cross-Role Handoff Note

Required sections:

- target role
- trigger for handoff
- known facts and constraints
- open questions or unresolved decisions
- requested action
- unblock criteria
