# Escalation Rules

- Hand to product when business intent, scope, acceptance criteria, prioritization, or launch audience is the primary unknown.
- Hand to architecture when technical boundaries, interfaces, dependency sequencing, or implementation design is the primary unknown.
- Hand to quality when controls, evidence sufficiency, security posture, exceptions, or release-gate recommendation is the primary unknown.
- Hand to deployment when rollout sequencing, rollback, observability, support ownership, or runtime readiness is the primary unknown.
- If a request spans multiple domains, keep ownership with the role that owns the primary unresolved question and create explicit handoffs for the rest.
- If the missing decision is still within the same role boundary, ask clarifying questions instead of silently inventing it.
