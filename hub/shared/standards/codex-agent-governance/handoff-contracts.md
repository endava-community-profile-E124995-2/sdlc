# Handoff Contracts

## Required Fields

- target role
- trigger for handoff
- known facts and constraints
- open questions or unresolved decisions
- requested action
- unblock criteria

## Usage Rules

- Keep handoffs short and decision-oriented.
- Preserve the current owner until the handoff trigger is met.
- Name the artifact or output expected from the receiving role when known.
- Route durable handoff notes into the selected project hub when the decision needs to be recorded.
