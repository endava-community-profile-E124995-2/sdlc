# Requirements Workflow

## Required Inputs

- discovery brief or approved change request
- in-scope and out-of-scope boundaries
- business rules and acceptance criteria inputs

## Role Handoffs

- primary: Product and Requirements Manager
- downstream handoff: Architecture and Engineering Manager

## Expected Artifacts

- PRD or requirements set
- acceptance criteria
- dependency and risk summary

## Output Location

- `projects/<project-id>/hub/product/`
- `projects/<project-id>/hub/decisions/` when requirement trade-offs need an explicit decision memo
