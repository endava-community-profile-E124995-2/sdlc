# Intake Workflow

## Required Inputs

- request source or stakeholder
- target project id or justification for shared-hub-only work
- short problem statement

## Role Handoffs

- primary: Product and Requirements Manager
- optional handoff: Architecture and Engineering Manager when feasibility is immediately blocking intake

## Expected Artifacts

- intake note
- initial routing decision
- open questions list

## Output Location

- `projects/<project-id>/hub/product/` for project-specific intake
- `hub/shared/portfolio/` for workspace-level intake or portfolio triage
