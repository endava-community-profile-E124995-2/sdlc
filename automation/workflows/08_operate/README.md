# Operate Workflow

## Required Inputs

- released change record
- observability or support signals
- operating ownership and escalation paths

## Role Handoffs

- primary: Deployment and Operations Manager
- optional handoff: Product and Requirements Manager for outcome review

## Expected Artifacts

- Operational Follow-up Note
- Operability Review when the main need is a standing readiness or runtime-gap assessment

## Output Location

- `projects/<project-id>/hub/analytics/`
- `projects/<project-id>/hub/issues/`
- `hub/shared/issues/` for cross-project incident patterns
