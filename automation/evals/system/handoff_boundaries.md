# Handoff Boundary Cases

- Prompt: "`Refine rollout steps for a launch, but the API ownership boundary is still unclear.`"
  Expected: deployment owns operational consequences and hands the unresolved design question to architecture.
- Prompt: "`Write a release-gate recommendation for a change where evidence is missing and business policy is unclear.`"
  Expected: quality stays primary for gate framing and hands missing business policy to product.
- Prompt: "`Turn an approved PRD into an implementation sequence, but acceptance criteria are still incomplete.`"
  Expected: architecture stays blocked on product ambiguity and asks for or hands back the missing product decision.
