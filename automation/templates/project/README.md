# Project Template

Use this template when onboarding a new attached project.

Required committed artifacts:

1. `projects/<project-id>/project.yaml`
2. a project entry in `projects/registry.yaml`
3. the project hub skeleton under `projects/<project-id>/hub/`

Optional local artifact:

- `projects/<project-id>/attachment.local.yaml`, created from `attachment.local.yaml.example` and kept untracked
