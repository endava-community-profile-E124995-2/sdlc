# Context Variant: Startup

## Use When

Use when the team moves quickly, role boundaries are light, and the release path needs to stay concise without losing owner clarity.

## Overrides

- keep artifacts short but still name owners, checkpoints, and rollback expectations
- prefer the smallest viable rollout sequence that still exposes hold points
- make support ownership explicit because it is often informal in startup environments

## Risks To Emphasize

- `risk.rollout_coordination_drift`
- `risk.observability_blind_spot`
