# Artifact Contract: Discovery Brief

```yaml
id: artifact.discovery_brief
artifact_name: Discovery brief
purpose: Frame a problem worth investigating and define the evidence work needed before commitment.
required_sections:
  - problem_statement
  - target_user_or_segment
  - evidence_snapshot
  - assumptions
  - open_questions
  - next_validation_step
optional_sections:
  - urgency
  - constraint_notes
  - expected_business_outcome
minimum_quality_bar: A serious brief names the user, problem, current evidence, top assumptions, and the next validation step.
common_failure_modes:
  - solution is described before the problem is clear
  - no evidence source is named
  - assumptions are hidden inside claims
  - the next validation step is missing
good_example_ref: corpus/examples/discovery_brief_good_b2b_workflow.md
bad_example_ref: corpus/examples/discovery_brief_bad_feature_factory.md
```

## Template

1. Problem statement
2. Target user or segment
3. Evidence snapshot
4. Assumptions and unknowns
5. Why this matters now
6. Next validation step
