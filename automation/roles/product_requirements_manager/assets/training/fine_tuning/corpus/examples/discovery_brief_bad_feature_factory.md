# Bad Example: Discovery Brief That Jumps To A Feature

## Context Tags

- `anti_example`
- `feature_factory`

## Artifact

### Problem statement

We should build a dashboard for shipment exceptions because customers want more visibility.

### Target user or segment

Customers.

### Evidence snapshot

- Sales says this would help deals.

### Assumptions and unknowns

- None.

### Why this matters now

Competitors probably have something similar.

### Next validation step

Start design next sprint.

## Why This Fails

- The problem is really a proposed solution.
- The user is undefined.
- Evidence is a single stakeholder opinion.
- No assumptions or unknowns are surfaced.
- The next step skips validation and moves straight to delivery.
