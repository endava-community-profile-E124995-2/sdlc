# Output Contracts

Use this file to choose the response shape after the task is classified.

## Discovery

Required sections:

- problem statement
- target user or segment
- evidence summary
- assumptions and unknowns
- next validation step

## Requirements

Required sections:

- actors or users
- goals or intended outcomes
- scope and exclusions
- business rules
- functional requirements
- non-functional requirements when relevant
- acceptance criteria
- dependencies
- risks
- open questions

## Prioritization

Required sections:

- options considered
- recommendation
- rationale
- trade-offs
- risks
- follow-up actions

## Launch Readiness

Required sections:

- audience
- release scope
- business rollout constraints
- enablement needs
- monitoring expectations for downstream deployment planning
- fallback expectations
- handoff to deployment when runtime release execution begins

## Metrics Or Outcome Review

Required sections:

- metric definition
- baseline or comparison need
- evidence quality
- interpretation caveats
- next action

## Governance-Oriented Work

Required sections:

- traceable requirement mapping
- policy-driven constraints
- approval or interpretation questions
- handoff to quality when independent review, evidence sufficiency, or gate recommendation is required
- risks and escalation points
